How to use git and github?

发表于2024-08-01|更新于2024-08-01|Pure EnglishLearning note
|字数总计:781|阅读时长:4分钟|阅读量:

Basic Concepts

repository：One project corresponds to one repository.The repository stores all the project codes.

star: If you think that project is useful or interesting, you can star it.On the one hand, that project will be more and more popular, on the other hand it’s more convenient for you to find the project.

fork: It means copy the project’s code in your repository which will have a mark “forked from xxxx”

Pull Request: Someone forked your code and add something new, they want to add their changes into your raw projects.Then they will send a PR(pull request) to you.You should review their code and do some tests, if you think there is no problem, you can merge all codes.

Issue: If you find some bugs or problems, you can raise in issue.

Basic Operation

add file、change file、delete file、search file(t): excute correspond operation and commit changes.You can find all the commits in the upper right corner.

new a repository、delete a repository(in the setting)、fork a repository. You can also propose a pull request or raise a issue to participate contributions.

Git Operation

Git working space: repository、staging area、working directory

  • The reason why we need staging area instead of only repository is that we need somewhere to choose which file we want to track which not,like some logs file or compiled file(.o file) etc.

Command

Basic command

  • git help : you can get some help on specific command.

  • git log --all --graph --decorate: you can get all the commits in a beautiful format.

  • config: git config --global user.name “xxx”、git config --global user.email “xxx@email.com”

    • With this configuration, github will display the information when you commit sth.
    • git config --list: get all the configuration informations.

  • git init: init a local repository with a .git file.After that you will get a (master) branch.

  • git add xxx.txt: Add some files.Pushing the xxx.txt into staging area.

  • git status: view current status.

  • git commit -m “some explainations”: commit the files in staging area to repository.

  • git rm xxx.txt: remove the xxx.txt from repository.

  • git checkout file.txt: drop the changes in your work directory.

    • git checkout hash value: switch the Head pointer.

  • git diff: you can get the differences between current directory and the commit version.

  • git cat-file -p hash value: look at the detail of the object.

Branch related

  • git branch: list all your branches.

    • git branch -vv: list more details.
    • git checkout -d “branch name”: build a new branch and switch into that branch.

  • git checkout master; git merge “other branch”: merge the branches and deal with conflicts.

Remote repository

  • git remote

Git underlevel

What’s Git?

Git is a sort of version control system.With git, you can track all your changes.At the same time you can cooperate with others in coding a same project.The basic unit is a snapshot which has tree and blobs.The whole git is composed of directed acyclic graph which is a number of snapshots.What we operate is all the references and objects.😄

The core of understanding git.First you should know every commit is a snapshot which consists of code 1. Second you should know what you operate is the object which consists of code 2. At last you should know how can we save all the snapshots effectively which consists of code 3.Building the relation of reference and objects.

Git中的对象object = blob | tree | commit
