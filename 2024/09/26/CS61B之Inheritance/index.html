Inheritance I: Interface and implementation Inheritance

当我们实现了两种列表且要使用两种列表来实现同一个功能，例如求一段文本中的最长字符个数，此时我们新建一个WordUtils类，并在类中新建longest方法，然后传入两种List，这种方式叫做重载(overloading)虽然实现了上述功能，但是如果再来一个列表，我们不得不再次重载，这里会发现代码重复度很高，而且如果longest有什么问题需要修改部分很多，维护成本很高。此时我们就可以利用接口继承，然后将参数改为父类参数即可。

接口继承：定义一个父类接口，在该接口中只声明各种方法不实现，如果实现该方法需使用关键词default，然后各个子类利用关键词implements去实现接口中的属于自己特定类的方法，在这里要配合标签@override，一方面能提醒你在实现接口中的方法，另一方面能帮助你检查一些拼写错误。

另一个值得注意的地方是接口继承是多代继承，这意味着你不仅继承父类，还会继承父类的父类，一直到原始祖先。

关于黄金等号原则(GRoE)，虽然黄金等号原则只是复制盒子里的值，特定类型的盒子只能装特定类型的值，但是接口继承中，父类盒子也能装子类值。Java中的静态类型和动态类型，静态类型就是我们声明时指定的类型，动态类型就是我们new出来时的类型，调用方法时是根据动态类型来调用，这就是为什么父类接口提供了默认方法，如果我们在子类中重写了该方法，调用时会调用该子类方法的原因。但是这有一种特殊的情况：当出现函数重载时，Java会根据传进参数的静态类型来决定调用哪个函数。

Inheritance II: Extends,Casting,Higher Order Functions

Extends和implements两者区别：一个是继承类，一个是继承接口。extends继承父类后可以调用父类的所有方法，包括构造方法，当我们在这个子类中添加构造方法时，Java会默认给我门调用父类的无参构造方法，如果要使用父类的有参构造方法需手动调用例如super(x)

特别强调 继承属于is a的关系，这意味着一旦你使用了extends，必须是猫是动物，狗是动物，而不是猫是爪子等等类似，但是猫有爪子，我们可以用has a的关系实现，你可以在爪子这个类中引入猫，并利用猫的有关爪子的方法，这就够了。一旦你产生继承关系，这意味着你将继承所有的父类方法，That’s dangerous!

另外，我们定义的所有的类都继承自Object，这意味着我们会继承所有的Object方法，其中典型的如toString()、equals()、hashcode()等等

封装(Encapsulation)，面向对象编程的又一个核心概念，目的就是为了降低复杂性，当我们思考一个很复杂的内容时，如果事事到细处，那这个事情将变得非常复杂，由此我们采用模块化和封装思想，即它的底层实现我们给隐藏起来，不允许访问，只暴露出一些功能接口，用户不需知道具体实现过程是怎样的，只要知道这个公共接口起到什么作用即可。然而，封装有时会被继承破环，如下在调用barkMany时出现由于重写了barkMany方法，导致无法实现原功能。

类型转换(Casting)，Java编译器在编译时是直接检查它的静态类型或者是声明类型，编译器在类型转换时会检查右边类型是否是左边类型，当然，继承的子类肯定是父类而父类不一定是子类，如果遇见不确定的情况，Java就会报错。为了避免报错，我们可以强制转换，此时相当于告诉编译器，你不用检查了，我确定它就是这个类型，然后Java在根据你告诉它的类型继续检查，左边是否等于右边。但是这里如果两个类型毫不相关，强制转换也会报错。

Important

@Compile Time

  1. 检查变量赋值是否有效

  2. 检查方法调用是否有效（只考虑静态类型和静态类型的父类们）

@Run Time

  1. 检查是否有重写方法

  2. 确保转换后的对象能够赋值给他们的变量

注意：

  1. 左边是静态类型右边是动态类型，实际中运行时看的是动态类型，而编译时看的是静态类型。这意味着我们实际中调用方法时取决于动态类型是什么来调用其方法，而不是根据声明的类型来调用其方法。

  2. 在检查时是先看静态类型满不满足要求，一旦有满足要求的则进行方法的绑定，如果涉及到动态类型的方法选择，那么看是否有重写我们所绑定的方法，如果没有则按原来执行，如果有则执行新的。

  3. 涉及到类型转换时，类型转换相当于左边的类型声明，最终动态类型相当于我们右边new的内容

即 Animal a = new Dog(); => (Animal) (new Dog())

高级函数，就是将函数作为一个参数传递到一个函数中。在python中使用高阶函数非常简单，只需要定义一个函数然后传进去就完事了，Java中同理，但是Java中传递参数需要指定类型，于是我们必须给这个函数指定一个类型，结和Java的面向对象特性，可以是一个类（引用）。

Inheritance III: Subtype Polymorphism, Comparators, Comparable

多态(polymorphism)：是指同一个方法在不同对象上具有不同的行为。通过多态，程序可以在运行时决定调用哪个方法，从而提高代码的灵活性和可扩展性。
